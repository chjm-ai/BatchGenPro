# Video Task Async Design

**目标**

把 `/api/batch/generate-video` 从“同步执行后再返回”改成真正的异步任务提交接口，避免线上在长视频生成时被前端或 Nginx 的 300 秒超时截断。

**范围**

- 后端视频任务创建接口改为立即返回 `task_id`
- 实际视频生成放入后台线程执行
- 后台线程复用现有 `process_video_generate_sync`
- 前端提交成功提示改成“任务已提交”，继续依赖现有任务轮询拿结果
- 增加后端回归测试，锁定“接口秒回 + 已调度后台任务”的行为

**不做**

- 不切换到 Celery worker
- 不重构现有 Redis 任务结构
- 不改图片批量生成链路
- 不新增前端测试框架

**实现方案**

在 `backend/app.py` 中提取一个启动后台视频任务的辅助函数。路由完成参数校验、创建任务、更新为 `processing` 后，直接调用该辅助函数启动守护线程并立即返回。线程内部执行现有同步生成逻辑，并在成功或失败后更新任务状态。这样前端能稳定拿到 `task_id`，后续由现有 `BatchTaskManager` 轮询状态和结果。

**错误处理**

- 后台线程捕获所有异常并把任务标记为 `failed`
- 路由层只处理参数错误和任务创建失败
- 线程启动失败时，路由返回 500，并将任务标记为 `failed`

**验证**

- 后端单元测试验证视频创建接口会调用后台调度函数并立即返回 200
- 后端既有视频生成测试继续通过
- 前端构建通过，确保提示文案改动无语法问题
